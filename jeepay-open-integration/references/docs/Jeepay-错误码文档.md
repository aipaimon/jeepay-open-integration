## 一、系统统一错误码（ApiCodeEnum）

接口返回结构：`{code, msg, data, encryptData, sign}`

| 错误码 | 枚举名 | 含义 | 处理建议 |
|--------|--------|------|----------|
| 0 | `SUCCESS` | 请求成功 | - |
| 10 | `SYSTEM_ERROR` | 系统异常 | 联系技术支持 |
| 11 | `PARAMS_ERROR` | 参数有误 | 检查请求参数 |
| 12 | `DB_ERROR` | 数据库服务异常 | 联系技术支持 |
| 5000 | `SYS_OPERATION_FAIL_CREATE` | 新增失败 | 检查参数后重试 |
| 5001 | `SYS_OPERATION_FAIL_DELETE` | 删除失败 | 检查记录是否存在 |
| 5002 | `SYS_OPERATION_FAIL_UPDATE` | 修改失败 | 检查记录是否存在 |
| 5003 | `SYS_OPERATION_FAIL_SELETE` | 记录不存在 | 检查查询条件 |
| 5004 | `SYS_PERMISSION_ERROR` | 权限错误 | 检查账号权限 |
| 5005 | `SYS_USER_PWD_EXPIRED` | 用户密码已过期 | 修改密码 |
| 5006 | `SYS_USER_UN_AUDIT` | 用户未通过审核 | 提交审核资料 |
| 9999 | `CUSTOM_FAIL` | 自定义业务异常 | 查看 msg 获取具体原因 |

## 二、渠道状态枚举（ChannelRetMsg.ChannelState）

用于描述与第三方支付渠道交互的结果：

| 枚举值 | 含义 | 处理建议 |
|--------|------|----------|
| `CONFIRM_SUCCESS` | 明确成功 | 订单处理完成 |
| `CONFIRM_FAIL` | 明确失败 | 订单处理失败 |
| `CONFIRM_FAIL_WAIT_RETRY` | 明确失败但可重试 | 可重新发起 |
| `WAITING` | 处理中 | 等待回调或主动查询 |
| `WAITING_AUTH` | 处理中，需要验证 | 完成验证后继续 |
| `UNKNOWN` | 状态未知 | **必须调用查单接口确认** |
| `API_RET_ERROR` | 渠道侧异常 | 联系渠道方 |
| `SYS_ERROR` | 系统异常 | 联系技术支持 |

## 三、常见业务错误消息（code=9999）

### 3.1 支付 API 网关类

| 错误消息 | 原因 | 解决方案 |
|----------|------|----------|
| 参数有误！ | 请求参数错误 | 检查必填参数和格式 |
| 通信秘钥错误！ | appSecret 不正确 | 检查商户配置的 appSecret |
| 商户不存在，请检查商户号 | mchNo 不存在 | 检查商户号是否正确 |
| 商户[X]信息不存在或商户状态不可用 | 商户被禁用 | 联系运营平台开通 |
| 商户应用[X]不存在或应用状态不可用 | appId 不存在或被禁用 | 检查应用配置 |
| 参数appId与商户号不匹配 | appId 和 mchNo 不匹配 | 检查参数对应关系 |
| 商户未配置签名方式 | 未配置签名类型 | 在商户中心配置 |
| 商户不支持[X]签名方式 | 签名方式不支持 | 更换签名方式 |
| 验签失败 | 签名验证失败 | 检查签名算法和参数 |
| 不支持的签名方式 | signType 不支持 | 使用 MD5 |
| 当前服务商状态不可用 | 服务商被禁用 | 联系运营 |
| 代理商(X)无此接口[Y]权限！ | 接口权限不足 | 联系开通接口权限 |
| 商户(X)无此接口[Y]权限！ | 接口权限不足 | 联系开通接口权限 |
| 当前代理商与商户号关系不匹配 | 代理商和商户无绑定关系 | 检查代理关系 |

## 四、签名验签规范

### 4.1 支持的签名方式

| 签名方式 | 说明 | 推荐度 |
|----------|------|--------|
| MD5 | 使用 appSecret 进行 MD5 签名 | 推荐 |

### 4.2 签名参数排序

所有请求参数按 key 的 ASCII 码升序排列，拼接后进行签名。

### 4.3 异步通知验签

收到异步通知后**必须先验签**，确保通知来自 Jeepay。

## 五、错误处理最佳实践

### 5.1 下单接口错误处理

```
1. 检查 code 是否为 0
2. 如果 code != 0，查看 msg 获取错误原因
3. 如果 code = 0，检查 sign 验签
4. 验签通过后，根据业务逻辑处理
```

### 5.2 异步通知处理

```
1. 收到通知后先验签
2. 验签失败直接返回失败
3. 验签通过后处理业务逻辑
4. 处理成功返回 "success"
5. 处理失败返回具体错误信息
```

### 5.3 状态未知处理

```
1. 当返回状态为 UNKNOWN 时
2. 必须调用查单接口确认订单状态
3. 根据查单结果决定后续处理
4. 切勿直接判定为成功或失败
```

## 六、Jeepay 开源版特有错误处理

### 6.1 部署配置相关错误

| 错误场景 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 无法连接支付网关 | 网关地址配置错误 | 检查网关地址配置 |
| 数据库连接失败 | 数据库配置错误 | 检查数据库连接配置 |
| Redis连接失败 | Redis配置错误 | 检查Redis连接配置 |
| 支付渠道不可用 | 渠道参数未配置 | 在运营平台配置渠道参数 |

### 6.2 商户管理相关错误

| 错误场景 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 商户不存在 | 商户未入驻 | 在运营平台创建商户 |
| 应用不存在 | 应用未创建 | 在商户中心创建应用 |
| 商户被禁用 | 商户状态异常 | 联系运营平台处理 |
